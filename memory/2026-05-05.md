### Secrets/ENV Architecture Explained
- Workspace `.env` is for Python scripts only (`tools/llm_router.py`), loaded via `python-dotenv`
- OpenClaw does NOT read `.env` files — its keys are in `~/.openclaw/openclaw.json` under `models.providers.*`
- OpenClaw supports **SecretRefs** for referencing secrets without plaintext:
  - `source: "env"` — references real shell environment variables (set before gateway starts)
  - `source: "file"` — reads from a JSON file (e.g. `~/.openclaw/secrets.json`)
  - `source: "exec"` — runs an external command (1Password, Vault, sops, etc.)
- Recommended setup: `openclaw secrets configure` wizard, or `source: "file"` pointing to `~/.openclaw/secrets.json`
- Secrets are resolved eagerly at startup into an in-memory snapshot, not lazily per-request
- Failed resolution on active surfaces = gateway won't start (fail-fast)

### Key Takeaway
- Don't use `.env` for OpenClaw config — use SecretRefs
- Two secrets providers configured in workspace `.env`: DeepSeek (`sk-256...`) and ZLM (`657f634...`)
- These need to be either in shell env vars or a secrets.json for OpenClaw to use them
